===============================
SYSTEM ARCHITECTURE & FEATURES:
===============================

1. The Assembler (asm.cpp):
    - Two-Pass Architecture:
        . Pass 1: Performs syntax analysis, strips comments, validates label naming, and populates the symbol_map with PC addresses.
        . Pass 2: Resolves operands, calculates PC-relative offsets for branch instructions (call, brz, brlz, br), and generates machine code.
    - Data Structures: Uses std::map for the Instruction Set Architecture (ISA) lookup, symbol mapping, and an error registry to track faults by line number.
    - Error Detection: Successfully detects:
        . Duplicate labels, Invalid label syntax (non-alphanumeric), Unrecognized mnemonics, Missing or extra operands, Undefined symbols.
    - Output Files: Generates a machine-readable object file (.o), an advanced listing file (.lst) showing addresses and hex code, and a comprehensive log file (.log).
    - Can implements partially the instruction SET.

2. The Emulator (emu.cpp):
    - Memory Management: Simulates a 10,000-word memory system and standard registers (A, B, PC, SP).
    - Execution Engine: Implements the full range of instructions, including arithmetic, stack operations, and branch logic.
    - Interactive Flags: 
        . -t: Provides a full execution trace.
        . -b: Produces a memory dump before execution.
        . -a: Executes the program and produces a memory dump after execution.
        . -ISA: Displays the supported instruction set.
    - Safety Features: Detects stack overflows, infinite loop traps (branch to self), and unrecognized opcodes.
